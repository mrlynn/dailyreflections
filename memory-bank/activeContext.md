## Implemented Decisions

### Data Fetching Strategy
**Decision**: Client components with fetch for all data
**Rationale**: Better for dynamic content, optimistic updates

### Component Structure
**Decision**: Separate components - ReflectionCard, CommentList, CommentItem, CommentForm
**Rationale**: Clear separation of concerns, reusable, testable ✅

### Date Handling
**Decision**: Use "MM-DD" format as dateKey throughout
**Rationale**: Simple, consistent, easy to query/index ✅

### Theme Design
**Decision**: MongoDB-inspired color palette with Material UI
**Rationale**: Professional, modern, aligned with user preferences ✅
